# DotVoice

Real-time, camera-based Braille reader that converts physical raised-dot Braille into English text and spoken audio — fully offline, eyes-free, and accessible.

## The Problem

Blind and low-vision users rely on Braille, but pointing a camera at a Braille page unseen is nearly impossible without sighted help. DotVoice solves this with **spoken camera guidance** — the app talks you through positioning the camera so you never need to see the screen.

## Features

- Reads REAL physical handwritten Braille from camera — validated on the organizer's own handwritten test image
- Real-time dot detection and Grade-1 Braille decoding
- Spoken camera guidance: "Move closer", "Hold steady", "Braille detected, reading now"
- Speech output: browser Web Speech API (web app) and offline `pyttsx3` (CLI)
- Accessible web UI: high-contrast, large text, ARIA live regions, keyboard shortcuts
- Multi-frame voting for stable readings
- Perspective correction and adaptive lighting normalization
- 100% character accuracy on clean synthetic test set (15 words)
- **75.8% pass rate (91/120)** across a 120-scenario robustness harness (noise, rotation, blur, uneven lighting)
- ~25–30 FPS camera capture with Braille processing every 5th frame (real-time feedback)

## How It Detects Physical Braille

1. **Preprocess:** Each camera frame is converted to grayscale, perspective-corrected, lighting-normalized (divide by large-kernel blur), denoised, and CLAHE-enhanced to make raised dots pop.
2. **Detect dots:** `cv2.SimpleBlobDetector` finds small, circular, dark blobs. A connected-components fallback handles edge cases.
3. **Segment grid:** The median nearest-neighbor distance between dots estimates the base unit `u`. Dots are clustered into columns and rows, then split into 2×3 Braille cells using the physical geometry ratio (inter-cell gap ≈ 2.4·u).
4. **Decode:** Each cell's dot set (positions 1–6) is looked up in the Grade-1 Braille table, with a state machine handling capital and number prefixes.
5. **Vote:** The last 5 decoded strings are majority-voted to eliminate single-frame errors.

## Tech Stack

- Python 3.10+
- Classical, deterministic CV pipeline — no trained model or weights required
- OpenCV — image processing, dot detection, deskew
- NumPy / SciPy / scikit-image — numerical and spatial processing
- Flask — web server, MJPEG camera stream, Server-Sent Events
- pyttsx3 — offline text-to-speech
- Web Speech API — browser-side TTS
- pytest — unit tests + scenario pass-rate harness

## Setup

```bash
git clone https://github.com/Dxrshn/dot-voice.git
cd dot-voice
python -m venv venv
source venv/bin/activate
pip install -r requirements.txt
```

**Web UI (recommended):**
```bash
python app.py
```
Open `http://localhost:5001`

**CLI:**
```bash
python run_cli.py
```

## Accuracy

Three independently measured tiers — all reproducible, no cherry-picking.

| Tier | Result |
| --- | --- |
| Unit tests (pytest) | 7/7 pass |
| 120-scenario robustness harness | 75.8% (91/120) |
| Real handwritten Braille (data/real/joel.png) | 5/5 lines located, 4/7 words exact |

On the real handwritten test image, "jaihind", "india", "visually", and "great" decode exactly.
Remaining errors are single-dot row slips on warped handwritten input; "sciobraille" likely uses a
Grade-2 contraction (a stated Grade-1 limitation). Weakest synthetic condition is rotation >=10 degrees;
small tilts, noise, and uneven lighting are handled well.

## Known Limitations

- Grade-1 Braille only (no contracted Grade-2 yet)
- Real-world accuracy depends heavily on lighting angle — raking side-light works best
- Physical Braille samples must have clearly raised dots (embossed paper or puffy paint)

## Future Improvements

- Optional ML-based dot detector for harsh lighting (current pipeline is classical and fully reproducible)
- Add Grade-2 contracted Braille support
- Mobile camera support via browser `getUserMedia`
- Multi-line Braille page reading
